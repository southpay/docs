## How webhooks work

1. You register an HTTPS endpoint and specify which event types to subscribe to
2. Southpay generates a signing secret and returns it **once** — store it immediately
3. When an event occurs, Southpay `POST`s a signed JSON payload to your endpoint
4. Your server verifies the signature, returns `2xx`, and processes the event
5. If delivery fails, Southpay retries automatically up to 10 times

---

## Register an endpoint

```bash
curl -X POST https://api.southpay.io/api/v2/webhook_endpoints \
  -H "Authorization: Bearer sp_test_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "webhook_endpoint": {
      "url": "https://yourapp.com/webhooks/southpay",
      "description": "Order fulfillment handler",
      "subscribed_events": [
        "payment_intent.completed",
        "payment_intent.failed",
        "payment_intent.expired"
      ]
    }
  }'
```

**Response:**

```json
{
  "id": "018f...",
  "url": "https://yourapp.com/webhooks/southpay",
  "status": "active",
  "subscribed_events": [
    "payment_intent.completed",
    "payment_intent.failed",
    "payment_intent.expired"
  ],
  "signing_secret": "whsec_a1b2c3d4e5f6...",
  "created_at": "2026-04-19T14:00:00.000Z"
}
```

<Warning>
  The `signing_secret` is returned **only on creation and after secret rotation**. Store it in a secrets manager immediately. It cannot be retrieved again.
</Warning>

## Event payload structure

Every webhook is a `POST` with `Content-Type: application/json` and this envelope:

```json
{
  "id": "018f1a2b-ffff-7000-8000-000000000099",
  "type": "payment_intent.completed",
  "created": 1745071260,
  "livemode": false,
  "data": {
    "object": {
      "id": "018f1a2b-3c4d-7e5f-9a0b-1c2d3e4f5a6b",
      "reference": "A1B2C3D4E5F6G7H8",
      "status": "completed",
      "state_reason": "completed_exact",
      "settlement_currency": "USD",
      "settlement_amount": "149.00",
      "from_status": "processing",
      "to_status": "completed",
      "metadata": {
        "order_id": "order_8721"
      }
    }
  }
}
```

| Field | Type | Description |
| --- | --- | --- |
| `id` | string (UUID) | Unique event ID. Use for deduplication. |
| `type` | string | Event type (see [Event types](#event-types)) |
| `created` | integer | Unix timestamp (seconds) |
| `livemode` | boolean | `false` for test mode events |
| `data.object` | object | The resource that changed |

## Request headers

Every delivery includes:

| Header | Description |
| --- | --- |
| `Southpay-Signature` | Signature string: `t=<timestamp>,v1=<digest>` |
| `Southpay-Delivery-Id` | UUID of this specific delivery attempt |
| `Content-Type` | `application/json` |
| `User-Agent` | `Southpay-Webhooks/1.0` |

## Verifying signatures

**Always verify the signature before processing any event.** This proves the request came from Southpay and that the payload was not tampered with.

### How it works

1. Parse `Southpay-Signature` header into `t` (timestamp) and `v1` (hex digest)
2. Construct the signed string: `{timestamp}.{raw_request_body}`
3. Compute `HMAC-SHA256(signing_secret, signed_string)` as hex
4. Compare to `v1` using a **timing-safe equality check**

```
Southpay-Signature: t=1745071260,v1=a3f9d2e1b4c7...
signed_payload     = "1745071260.{raw_json_body}"
expected_digest    = HMAC-SHA256(signing_secret, signed_payload)
```

<Warning>
  Compute HMAC against the **raw request body bytes**, not a re-serialized JSON object. Parsing and re-stringifying JSON can alter whitespace or key order and will break verification.
</Warning>

### Verification examples

<CodeGroup>

```typescript Node.js
import { createHmac, timingSafeEqual } from "crypto";
import express from "express";

function verifySignature(rawBody: Buffer, header: string, secret: string): boolean {
  const parts = Object.fromEntries(
    header.split(",").map(p => p.split("=") as [string, string])
  );
  const timestamp = parts["t"];
  const v1 = parts["v1"];
  if (!timestamp || !v1) return false;

  const signedPayload = `${timestamp}.${rawBody.toString("utf8")}`;
  const computed = createHmac("sha256", secret)
    .update(signedPayload)
    .digest("hex");

  const a = Buffer.from(computed, "hex");
  const b = Buffer.from(v1, "hex");
  if (a.length !== b.length) return false;
  return timingSafeEqual(a, b);
}

const app = express();

app.post(
  "/webhooks/southpay",
  express.raw({ type: "application/json" }),
  (req, res) => {
    const sig = req.headers["southpay-signature"] as string | undefined;
    const secret = process.env.SOUTHPAY_WEBHOOK_SECRET!;

    if (!sig || !verifySignature(req.body as Buffer, sig, secret)) {
      return res.status(401).json({ error: "Invalid signature" });
    }

    const event = JSON.parse((req.body as Buffer).toString());

    switch (event.type) {
      case "payment_intent.completed":
        fulfillOrder(event.data.object);
        break;
      case "payment_intent.failed":
      case "payment_intent.expired":
        cancelOrder(event.data.object);
        break;
    }

    res.json({ received: true });
  }
);
```

```python Python
import hmac
import hashlib
import os
from flask import Flask, request, jsonify

app = Flask(__name__)

def verify_signature(raw_body: bytes, header: str, secret: str) -> bool:
    parts = dict(p.split("=", 1) for p in header.split(","))
    timestamp = parts.get("t")
    v1 = parts.get("v1")
    if not timestamp or not v1:
        return False

    signed_payload = f"{timestamp}.{raw_body.decode('utf-8')}"
    computed = hmac.new(
        secret.encode("utf-8"),
        signed_payload.encode("utf-8"),
        hashlib.sha256
    ).hexdigest()

    return hmac.compare_digest(computed, v1)

@app.post("/webhooks/southpay")
def webhook():
    sig = request.headers.get("Southpay-Signature", "")
    secret = os.environ["SOUTHPAY_WEBHOOK_SECRET"]

    if not verify_signature(request.data, sig, secret):
        return jsonify(error="Invalid signature"), 401

    event = request.get_json()
    event_type = event.get("type")

    if event_type == "payment_intent.completed":
        fulfill_order(event["data"]["object"])
    elif event_type in ("payment_intent.failed", "payment_intent.expired"):
        cancel_order(event["data"]["object"])

    return jsonify(received=True)
```

```go Go
package main

import (
    "crypto/hmac"
    "crypto/sha256"
    "encoding/hex"
    "encoding/json"
    "fmt"
    "io"
    "net/http"
    "os"
    "strings"
)

func verifySignature(rawBody []byte, header, secret string) bool {
    parts := make(map[string]string)
    for _, p := range strings.Split(header, ",") {
        kv := strings.SplitN(p, "=", 2)
        if len(kv) == 2 {
            parts[kv[0]] = kv[1]
        }
    }
    timestamp, v1 := parts["t"], parts["v1"]
    if timestamp == "" || v1 == "" {
        return false
    }

    signedPayload := fmt.Sprintf("%s.%s", timestamp, string(rawBody))
    mac := hmac.New(sha256.New, []byte(secret))
    mac.Write([]byte(signedPayload))
    computed := hex.EncodeToString(mac.Sum(nil))

    return hmac.Equal([]byte(computed), []byte(v1))
}

func webhookHandler(w http.ResponseWriter, r *http.Request) {
    rawBody, _ := io.ReadAll(r.Body)
    sig := r.Header.Get("Southpay-Signature")
    secret := os.Getenv("SOUTHPAY_WEBHOOK_SECRET")

    if !verifySignature(rawBody, sig, secret) {
        http.Error(w, `{"error":"Invalid signature"}`, http.StatusUnauthorized)
        return
    }

    var event map[string]interface{}
    json.Unmarshal(rawBody, &event)

    switch event["type"] {
    case "payment_intent.completed":
        fulfillOrder(event["data"])
    case "payment_intent.failed", "payment_intent.expired":
        cancelOrder(event["data"])
    }

    w.Header().Set("Content-Type", "application/json")
    w.Write([]byte(`{"received":true}`))
}
```

## Event types

### Payment Intent events

| Event | Triggered when |
| --- | --- |
| `payment_intent.created` | A new Payment Intent is created |
| `payment_intent.processing` | An on-chain transaction is detected |
| `payment_intent.completed` | Payment confirmed and credited |
| `payment_intent.failed` | Payment detected but cannot complete |
| `payment_intent.expired` | Intent timed out with no payment |
| `payment_intent.refunded` | Funds returned to sender |

### Transaction events

| Event | Triggered when |
| --- | --- |
| `transaction.confirming` | New transaction detected, waiting for confirmations |
| `transaction.confirmed` | Transaction reached required confirmation count |
| `transaction.failed` | Transaction failed on-chain |

### Payout events

| Event | Triggered when |
| --- | --- |
| `payout.created` | A payout is initiated |
| `payout.confirmed` | Payout transaction confirmed on-chain |
| `payout.failed` | Payout transaction failed |

## Delivery and retries

Southpay expects a `2xx` response within **30 seconds** of sending the request.

If your endpoint returns a non-`2xx` status, times out, or is unreachable, the delivery is retried with exponential backoff:

| Attempt | Delay after previous |
| --- | --- |
| 1 | 10 seconds |
| 2 | 30 seconds |
| 3 | 2 minutes |
| 4 | 5 minutes |
| 5 | 10 minutes |
| 6 | 30 minutes |
| 7 | 1 hour |
| 8 | 2 hours |
| 9 | 5 hours |
| 10 | 10 hours |

After 10 failed attempts, the endpoint is marked `error` and you receive an email notification. Use the dashboard to redeliver individual events or re-enable the endpoint.

**`4xx` responses (except `429`)** are not retried — they indicate a client error (bad signature, endpoint logic error). Fix the issue and use the redeliver endpoint.

## Rotate your signing secret

Rotate without downtime — the new secret takes effect immediately:

```bash
curl -X POST https://api.southpay.io/api/v2/webhook_endpoints/{endpoint_id}/rotate_secret \
  -H "Authorization: Bearer sp_test_YOUR_KEY"
```

Returns the new `signing_secret`. Update your environment variable and re-deploy.

## Deduplication

Southpay guarantees **at-least-once delivery** — a successful delivery may occasionally be sent more than once during retries. Use the `id` field to deduplicate:

```typescript
const eventId = event.id;
if (await isEventAlreadyProcessed(eventId)) {
  return res.json({ received: true }); // idempotent
}
await markEventProcessed(eventId);
await processEvent(event);
```

## Security checklist

- Always verify `Southpay-Signature` before processing
- Use timing-safe comparison (`timingSafeEqual`, `hmac.compare_digest`) — never string equality
- Read the raw request body before parsing JSON — do not re-serialize for signature verification
- Store the signing secret in an environment variable, never in source code
- Your endpoint must use **HTTPS**
- Return `2xx` quickly (< 30s) — offload heavy work to a background queue
- Implement deduplication using the event `id`
