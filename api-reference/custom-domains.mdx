title: "Custom Domains"
description: "Add and verify custom domains to white-label the Southpay hosted checkout under your own brand."
sidebarTitle: "Custom Domains"
---

<Warning>
  Custom domains require the white-label feature flag to be enabled on your store.
</Warning>

<Warning>
  All endpoints require authentication. See [Authentication](/authentication) for details.
</Warning>

Base URL: `https://api.southpay.io/api/v2`

---

## List custom domains

<CodeGroup>
```bash Request
GET /custom_domains
Authorization: Bearer <API_KEY>
```

```json Response
[
  {
    "id": "cd_abc123",
    "hostname": "pay.yourapp.com",
    "verification_status": "verified",
    "verified_at": "2026-04-05T10:00:00.000Z",
    "created_at": "2026-04-01T00:00:00.000Z"
  }
]
```
</CodeGroup>

---

## Get a custom domain

```bash
GET /custom_domains/:id
Authorization: Bearer <API_KEY>
```

---

## Add a custom domain

<CodeGroup>
```bash Request
POST /custom_domains
Authorization: Bearer <API_KEY>
Content-Type: application/json

{ "custom_domain": { "hostname": "pay.yourapp.com" } }
```

```json Response 201
{
  "id": "cd_abc123",
  "hostname": "pay.yourapp.com",
  "verification_status": "pending",
  "verified_at": null,
  "created_at": "2026-04-10T00:00:00.000Z"
}
```
</CodeGroup>

After adding a domain, add the required DNS records provided in the dashboard, then call the verify endpoint.

---

## Verify a custom domain

<CodeGroup>
```bash Request
POST /custom_domains/:id/verify
Authorization: Bearer <API_KEY>
```

```json Response
{
  "id": "cd_abc123",
  "hostname": "pay.yourapp.com",
  "verification_status": "verified",
  "verified_at": "2026-04-10T00:05:00.000Z",
  ...
}
```
</CodeGroup>

Returns `422` if DNS records are not yet propagated.

---

## Remove a custom domain

Returns `204 No Content`.

```bash
DELETE /custom_domains/:id
Authorization: Bearer <API_KEY>
```
