## PaymentIntent lifecycle

A PaymentIntent has a single `status` field that represents its lifecycle.

Always rely on `status` as the source of truth.

| Status | Terminal | Description |
| --- | --- | --- |
| `pending` | No | Created, awaiting customer payment |
| `processing` | No | Crypto received, blockchain confirmations in progress |
| `completed` | Yes | Required funds received and confirmed |
| `expired` | Yes | Payment window elapsed before funds were received |
| `failed` | Yes | Payment could not be completed (under/overpaid) |
| `refunded` | Yes | Funds were returned to the customer |

### status_reason

`status_reason` explains why the PaymentIntent reached its current state.

| Reason | Status | Description |
| --- | --- | --- |
| `pending_confirmations` | processing | Crypto received, confirmations pending |
| `completed_exact` | completed | Exact required amount received |
| `completed_overpaid` | completed | More than required amount received |
| `failed_underpaid` | failed | Insufficient funds before expiration |
| `failed_overpaid` | failed | Excess funds rejected |
| `expired_timeout` | expired | No funds received before timeout |

## Create a payment intent

<CodeGroup>
```bash Request
POST /payments
Authorization: Bearer <API_KEY>
Content-Type: application/json
Idempotency-Key: order_123  # optional

{
  "payment_intent": {
    "amount": "49.99",
    "currency": "USD",
    "metadata": {
      "order_id": "order_123"
    }
  }
}
```

```json Response 201
{
  "id": "pi_abc123",
  "reference": "tok_xyz789",
  "status": "pending",
  "state_reason": null,
  "settlement_currency": "USD",
  "settlement_amount": "49.99",
  "terminal": false,
  "checkout_url": "https://checkout.southpay.io/pay/tok_xyz789",
  "expires_at": "2026-04-10T13:00:00.000Z",
  "created_at": "2026-04-10T12:00:00.000Z",
  "payment_addresses": [...]
}
```
</CodeGroup>

**Request fields**

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `payment_intent[amount]` | string | Yes | Decimal amount (e.g. `"49.99"`) |
| `payment_intent[currency]` | string | Yes | ISO 4217 fiat code, uppercase (e.g. `"USD"`) |
| `payment_intent[metadata]` | object | No | Arbitrary key-value pairs attached to the payment |
| `Idempotency-Key` | header | No | Pass your order ID to safely retry without creating duplicates |

<Tip>
  Redirect your customer to `checkout_url` — no frontend integration required.
</Tip>
