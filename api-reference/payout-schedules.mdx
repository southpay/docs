## Create a payout schedule

<CodeGroup>
```bash Request
POST /payout_schedules
Authorization: Bearer <API_KEY>
Content-Type: application/json

{
  "payout_schedule": {
    "asset_id": "ETH_USDC",
    "frequency": "daily",
    "minimum_threshold": 1000000,
    "destination_address": "0xabc...",
    "next_payout_at": "2026-04-11T00:00:00.000Z"
  }
}
```

```json Response 201
{
  "id": "ps_abc123",
  "frequency": "daily",
  "minimum_threshold": 1000000,
  "destination_address": "0xabc...",
  "next_payout_at": "2026-04-11T00:00:00.000Z",
  "active": true,
  ...
}
```
</CodeGroup>

**Request fields**

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `payout_schedule[asset_id]` | string | Yes | Asset to pay out. See [Supported Assets](/api-reference/assets). |
| `payout_schedule[frequency]` | string | Yes | `daily`, `weekly`, or `monthly` |
| `payout_schedule[minimum_threshold]` | integer | Yes | Minimum balance (atomic units) before a payout triggers |
| `payout_schedule[destination_address]` | string | Yes | On-chain address to receive funds |
| `payout_schedule[next_payout_at]` | string | No | ISO 8601 datetime of the first payout. Defaults to next period. |

## Update a payout schedule

The `asset_id` cannot be changed after creation.

<CodeGroup>
```bash Request
PATCH /payout_schedules/:id
Authorization: Bearer <API_KEY>
Content-Type: application/json

{
  "payout_schedule": {
    "frequency": "weekly",
    "minimum_threshold": 5000000,
    "destination_address": "0xnew...",
    "active": true
  }
}
```

```json Response
{ "id": "ps_abc123", "frequency": "weekly", ... }
```
</CodeGroup>

## Delete a payout schedule

Deactivates the schedule (soft delete). Returns `204 No Content`.

```bash
DELETE /payout_schedules/:id
Authorization: Bearer <API_KEY>
```
